(Gitについて)
・リモート・ローカルリポジトリについて
リモートリポジトリ：専用のサーバーに配置して複数人で共有するためのリポジトリ
ローカルリポジトリ：自分のPC内でバージョン管理するために作成したリポジトリ
→双方の状態の同期を行うために、プッシュやプルを行う

・ワークツリー・インデックス・リポジトリについて
ワークツリー：Gitの管理下におかれた作業ディレクトリのこと
インデックス：ローカルリポジトリにコミットを行う準備をする場所(ステージングされる場所)
リポジトリ：ファイルやディレクトリの状態を記録する場所

(流れと関係性)
ワークツリーで作業・編集
↓
差分をインデックスにステージング
↓
コミットしてローカルリポジトリに変更履歴を記録
↓
プッシュしてリモートリポジトリと同期

・ブランチについて
→履歴の流れを分岐して記録していくもの
→→同じリポジトリ中で複数の変更を同時に進めていくために使用する

・プルリクエストについて
→開発者のローカルリポジトリでの変更を他の開発者に通知して、マージを依頼する機能
→→コードのレビューを受けることで、バグや記述ミスの発見に繋がってコードの品質を高める

・Gitコマンドについて
git branch：ローカルブランチの一覧を表示する
git branch -r：リモートブランチの一覧を表示する
git fetch：ローカルリポジトリ内のリモートブランチを最新化する
？：特定のローカルブランチに特定のリモートブランチの差分を反映する
？：特定のリモートブランチを元に新規ローカルブランチを作成する
git branch --contains：現在作業しているブランチを確認する
git checkout (ブランチ名)：作業するブランチを切り替える
git add (ファイル名)：コミット対象ファイル/ディレクトリをインデックスに追加する
git commit -m "(メッセージ)"：変更内容をコミットする
git push (リモートリポジトリ) (ブランチ名)：コミット内容をリモートリポジトリに反映する

(HTTPについて)
・HTTPについて
→Hyper Text Transfer Protocolの略
→→WWW(World Wide Web)上でWebサーバとクライアントが、 HTMLで書かれた文書などの情報をやりとりする時に使われる、通信手順(プロトコル)を意味する

・HTTPのリクエスト・レスポンスについて
(両者の流れ)
クライアント側のPCでブラウザにURLを入力
↓
クライアント側のPCからWebサーバーに「HTTPリクエスト」を送信
↓
Webサーバーが、「HTTPリクエスト」に対応する「HTTPレスポンス」をクライアント側のPCに送信
↓
クライアント側のPCでWebページを表示

クライアント：サービスを受ける側
サーバー：サービスを提供する側

・HTTPリクエストのGET・POSTについて
GET：HTTP通信において、サーバから情報を取得してくる時に使用
→送信できるデータ量に制限があり、テキストデータのみ送信できる
POST：HTTP通信において、サーバへ情報を登録する時に使用（データベースへの格納など）
→データ量が多い場合やバイナリデータを送りたい場合、パスワードなどの秘匿情報を送る場合など

(Laravelについて)
・.envファイルについて
→環境ごとに異なる設定をするためのファイル
→→開発環境と本番環境を切り替えたり、データベースなどの接続情報の変更を行うなど、Laravelにとって重要な設定変更を行うことができる

・アプリケーションキーについて
→暗号化の際に利用される、ランダムな英数字32文字

・artisanコマンドについて
→Laravelで使われるCLIの一つ
→→LaravelでWebサービスを作る際の設定やコードを簡単なコマンドで実装できる

・artisanコマンド例について
php artisan serve --port (ポート番号)：ポート指定でアプリケーションを起動
php artisan make:controller (コントローラ名)：コントローラークラスの生成
php artisan make:model (モデル名)：モデルクラスの生成
