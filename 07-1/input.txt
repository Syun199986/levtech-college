・バージョン管理システムについて
→特定のファイルに加えられていく変更履歴を記録・管理していくもの

・導入のメリットについて
以前の状態に戻せる：変更ファイルを過去の状態に復元したり、変更した内容を表示できる
変更履歴を全て記録できる：変更の履歴を遡って調査したり、特定部分の変更のみの取消ができる
デグレードの発生を防止できる：バグのある古いバージョンのソースコードを反映してしまうといったバージョン管理の失敗を防ぐことができる
※デグレード：新しいバージョンのソフトウェアが以前よりも品質を落としてしまうこと

・Gitについて
→分散型バージョン管理システム
→→上記のメリットや、ブランチを使ったバージョン管理、複数人で修正した部分を一つに統合することなどができる

・GitHubについて
→Gitの仕組みを利用して、全世界から自分の作品を保存、公開できるようにしたWebサービス
→→Web上でグラフィカルに扱える
→GitHubならではの機能を使える
→→フォーク、プルリクエスト、マージなど

・GitHubに反映させるまでの流れについて
サーバーでファイルを追加
↓
git addでステージング
↓
git commitでコミット(コミットメッセージを残す)
↓
git pushでリモートリポジトリに送る
↓
GitHubに反映
