・DBとその用途について
→決まった形式(データモデル)で整理されたデータの集まり
→集積したデータを抽出・編集・共有しやすくするなどの用途
→→必要な時に必要なデータをすぐに取り出せる

・RDBMSについて
→Relational DataBase Management System
リレーショナルデータベース：データベースのうち、データを表に似た構造で管理するもの

・Oracle・MySQL・PostgreSQL・SQL Serverについて
Oracle：世界中で利用者が多く、Win・Linux・Unixなど様々なプラットフォームに対応している
MySQL：オープンソースで無料で利用でき、OSに依存しないためほとんどのOSで利用できる
PostgreSQL：オブジェクト指向のプログラミング言語で操作・参照できる
SQL Server：Microsoft社が開発したRDBMSであり、WinOSとの相性が良い

・MariaDBについて
→MySQLのソースコードをベースにして、新機能追加やソースコードの改善が組み込まれたもの
→→クラスタ構成でのデータベース構築を手軽に行うことができる

・DBサーバーとDBクライアントについて
DBサーバー：アプリケーションで利用するデータを格納し操作するためのサーバー
DBクライアント：DBサーバーにネットワークを通じて接続するためのソフトウェア

・mysqlコマンドについて
mysql -u (ユーザー名)：ユーザーを指定して接続
mysql -u (ユーザー名) -p：ユーザーのパスワード認証で接続
use (DB名)：使用するDBを指定して接続

・DB操作用ターミナルでのコマンドについて
SHOW DATABASES：存在しているDBの一覧表示
SHOW TABLES：存在しているテーブルの一覧表示
USE (DB名)：データベースの切り替え
EXIT：ターミナルの終了

・文字コードについて
→コンピュータ上で文字を利用する目的で各文字に割り当てられるバイト表現
ASCII：半角英数字と記号のみを表現するために用いられる文字コード
Shift-JIS：ASCIIコードの文字に日本語の文字を加えた文字コード
EUC-JP：UNIX系のOSで日本語を表示する時に使用される文字コードのうちの一つ
UTF-8：ASCIIと互換性があり、ASCIIと同じ部分は1byteで表現、その他の部分を2~6byteで表現する可変長の符号化方式
